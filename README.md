## CS499 Capstone ePortfolio

Welcome to my ePortfolio. 
This portfolio demonstrates the culmination of my growth in software development throughout the Computer Science program through enhancements made to a contact management system across three key areas: software design, algorithms, and database integration.

## Professional Self-Assessment
Throughout my Computer Science program, I have developed a very strong foundation in software development, system design, and overall problem solving. After completing a large variety of different coursework, I have been able to learn a lot and really refine my technical abilities and my approach to building more practical, real-world applications. This portfolio reflects my growth from school projects with basic functionality to designing more structured, scalable, and maintainable systems that feel more purposeful and polished.

One of the most valuable aspects of this program has been learning how to approach problems to find innovative solutions and use good developer practices. Earlier on in my coursework and personal development, I typically just did whatever I could to get my code to function correctly, without thinking too much about the effects. Over time, I started to emphasize having clean structured code with strong separation of concerns that is maintainable and well-documented. This helps development to go much more smoothly, especially with big projects that evolve a lot over time and have high impact. This shift in my approach has been reflected in my work with the enhancements of the contact management system going from a simple in-memory unit test application to an interactive command-line interface with a persistent database.

Collaboration has also been an important component of my development. Reading and replying to others’ discussion posts throughout my courses have helped me to gain new perspectives on software development and learn about new concepts I might not have known before. These discussions also taught me how to communicate my technical ideas clearly and effectively so that when I am working with a team, I can ensure that we can collaborate without issue. Collaboration also means having good documentation and readable code so others can understand it. I have been able to strengthen my documentation skills and my approach to planning and structuring my code throughout this program so that I can be a better collaborator in the future. I have also shown that I can communicate my ideas effectively using diagrams such as UML, Sequence, and Class diagrams. This helps to plan effectively for a project and especially when working with a team so everyone can work towards a common goal.

From a software design perspective, I have gained a lot of experience designing modular systems with clearly defined responsibilities. I learned to separate business logic, data storage, and user interaction in distinct components to improve the maintainability and scalability of my applications. As projects grow, the need for this separation increases, so it’s important to have a strong foundation when structuring your classes. In this enhancement project, I heavily refactored the original code in order to separate the logic into a more organized architecture that made it much easier for me to build off of in the next enhancements.

My understanding of data structures and algorithms has significantly improved during the program as I have been able to explore more higher-level concepts and learn about different uses for different implementations. For example, in several courses I utilized hash maps to optimize data retrieval and implemented search and filtering features that are efficient with emphasis on the time complexity of each implementation. These experiences have taught me how to evaluate trade-offs between performance and complexity so that I could select the appropriate solution based on the circumstances of the application.

Database integration has been another big area of growth for me throughout this program. I developed a strong understanding of relational database concepts and how to use SQL effectively. In early courses or in my personal development, I typically only used in-memory storage when creating my applications, but this did not prove to be very useful. In many of my later courses I implemented MongoDB databases and learned a lot about CRUD functionality. In this project, I implemented a SQLite database and that helped me to think about how data is stored, retrieved, and managed, while also thinking about the connection handling and query design.

Security has also become an important part of my approach to software development. In this program I learned a lot about validating user input, handling exceptions properly without exposing internals, role-based access and authorization, encrypting data using different encryption keys, and managing vulnerabilities using tools like OWASP dependency check. This has helped me to develop a security first mindset where I think about security while working on everything because security should never be an afterthought.

The artifacts included in this portfolio all collectively demonstrate my growth across software design, algorithms, and databases. Rather than presenting unrelated projects, I chose to enhance a single artifact for each of these categories because in modern software development you typically have to utilize all of these skills for every application. This approach highlights my ability to iteratively improve a system, apply new concepts, and evaluate my design decisions over time. Each enhancement builds upon the previous one, which results in a more professional and complete solution. Overall, this portfolio not only enhances the technical skills that I have developed, but also my ability to learn and adapt as a developer and my readiness to contribute to real-world software projects.
