# Enhancement One - Software Design
The artifact I selected is a contact management system that I originally created in CS320 about 10 months ago. It is written in Java and was mostly used to get familiar with JUnit testing, so the program itself is simple and didn’t have any sort of user interface. It originally only included four classes: Contact, ContactService, ContactTest, and ContactServiceTest. The basic functionality allowed for creating, updating, and deleting contacts, but it was only accessible through the unit tests. 

I selected this artifact for my ePortfolio because I thought it had a good foundation and a lot of potential for me to showcase my understanding of key software development concepts with the enhancements. It focused on object-oriented design, input validation, and unit testing, which is integral to production-level code, but it lacked the functionality to really use it outside of just a school project. I was able to improve this artifact in several key areas for the software engineering and design category. Firstly, I introduced a new ContactStorage class that helped to separate the data management from the business logic in the ContactService class. This helped to improve the modularity of the code and made it easier to maintain. Secondly, I refactored a lot of the code in the Contact and ContactService classes to remove duplicate validation logic by introducing helper methods, as well as utilizing constants for the numbers used in those validations. This makes the classes much more readable and easier to update in the future. Another big update I made was the creation of the ContactApp class for the command-line interface (CLI). This allows users to interact with the application directly instead of relying on the unit tests. The CLI includes functionality for adding contacts, viewing all contacts, viewing a specific contact with more details, and deleting contacts. The CLI has validation loops to prevent runtime errors, and it improves user experience by allowing users to retry invalid inputs without stopping the application.

I believe I met the course outcomes through this enhancement. I demonstrated the ability to design and evaluate computing solutions by improving the applications structure by organizing and separating the classes better. I also showed my use of effective software engineering techniques by refactoring code, reducing duplication, and adding a command line interface to make the program more usable. My improvements to the code structure, readability, and comments show that I have strong professional communication skills and focus on good documentation. While this was an individual project, it also showcases my ability to collaborate effectively because my code updates are maintainable and readable for anyone that might look at my project in the future. I also demonstrated a security mindset as well by implementing consistent input validation to prevent invalid or malicious data. 

Throughout the process of enhancing this artifact, I learned the importance of designing software with scalability and maintainability in mind. This was a major reason I refactored the validation into helper methods with constant variables so that if I were to make updates to the validation in the future, I would only really need to update it in one central location, and the rest of the program would still function as intended. The main challenges I faced were with implementing that validation properly because I switched back and forth between different ways of organizing those methods. After a good amount of testing, I was able to zero in on a structure I liked and worked the best for my intentions.
